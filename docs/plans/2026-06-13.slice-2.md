# Excerpt-Anchored Sessions — Slice 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist the session anchor through the disk pipeline, feed it to the ACP prompt as first-message context, rewire ACP session init on activation, and remove the old "New Topic" path — so sessions survive restart.

**Architecture:** The anchor moves from the transient `Topic` merge into `PersistedTopic`; `topic-for-disk`'s strip-extra-keys allowlist then persists it with zero codec changes (IPC codecs already carry it via `Topic`). The anchor rides the first user message's `:context` (provenance, alongside excerpts) and renders main-side as an `Anchor:` section in the prompt body. `set-active` dispatches `:acp.effects/init-session` guarded by a "live this run" set so restored sessions resume exactly once.

**Tech Stack:** ClojureScript, Malli, Nexus, Electron. Strict FCIS — pure actions return effect descriptions; effects registered in `renderer/actions.cljs` / `main/actions.cljs`.

**Spec:** `docs/specs/2026-05-25-excerpt-anchored-sessions-design.md` (Slice 2 section)

**Decisions confirmed with Paul:**
- Persistence timing: keep current auto-save semantics (save on first message). Empty anchored shells deliberately don't survive restart.
- Topic→Session rename: deferred to its own pass. Not in this slice.
- `:anchor` is **required** in `PersistedTopic` (no unanchored sessions remain; clean-slate pre-release — old anchorless disk topics fail `topic-from-disk` and are skipped by `load-topics`' existing invalid-file handling).

**Verified facts:**
- `topic-to-ipc`/`topic-from-ipc` coerce against `Topic` → anchor already crosses IPC today; only the disk codec strips it.
- `:acp.effects/init-session` (renderer/actions.cljs:227) is registered but **never dispatched** — chat is dead on this branch ("app broken between slices," per spec). Task 7 revives it.
- `restore-with-topics` ordering is safe: Nexus runs `[:effects/save topics-path topics]` before `set-active`'s init-session effect reads `@store`, so resume-on-open sees the persisted session id.
- Helpers that exist and are reused: `topic-state/get-active-topic` (state/topic.cljs:15), `loading/loading?` (state/loading.cljs:7), `most-recent-anchored` auto-activation (already filters by `:anchor`).

(defn- excerpt-detail [{:keys [text locator]}]
  (let [{:keys [start-block]} locator]
    (str (locator-label locator) "\n"
         "      text:\n" (blockquote text) "\n"
         "      block context:\n" (blockquote (:block-text-snippet start-block)))))

(defn- render-excerpt [idx excerpt]
  (str "  [" (inc idx) "] " (excerpt-detail excerpt)))

(defn- render-anchor [anchor]
  (str "\nAnchor:\n  " (excerpt-detail anchor)))

  "Compose the prompt text body: user text, then the session anchor (first
   message only — the renderer gates injection), then a References section
   when the message carries excerpts."
  [{:keys [text context]}]
  (cond-> text
    (:anchor context)         (str (render-anchor (:anchor context)))
    (seq (:excerpts context)) (str (render-references (:excerpts context)))))

### Task 7: Rewire ACP init on activation, with a live-this-run guard

The ACP agent runs in-process with main, so a persisted `[:session :id]` must be resumed once per app run; within a run, re-activations (bar clicks) must not re-init. Guard = set of topic ids made live this run, plus the existing `loading?` flag to absorb double-clicks mid-init. `session-error` does **not** mark live, so re-activating retries.

New `src/gremllm/renderer/state/acp.cljs`:

```clojure
(ns gremllm.renderer.state.acp
  "Tracks which topics have a live ACP session this app run.
   Persisted session ids outlive the in-process agent, so each run must
   resume (or create) a session exactly once per topic.")

(def live-topics-path [:acp :live-topics])

(defn live? [state topic-id]
  (contains? (get-in state live-topics-path #{}) topic-id))

(defn mark-live [state topic-id]
  (conj (get-in state live-topics-path #{}) topic-id))
```

In `src/gremllm/renderer/actions/acp.cljs` (require `gremllm.renderer.state.acp :as acp-state` and `gremllm.renderer.state.loading :as loading-state`):

```clojure
(defn init-session-plan
  "Decide how to bring a topic's ACP session live: resume a persisted session
   id, create a new session, or nil when already live or init is in flight."
  [state topic-id]
  (when-not (or (acp-state/live? state topic-id)
                (loading-state/loading? state topic-id))
    (if-let [acp-session-id (topic-state/get-acp-session-id state topic-id)]
      [[:acp.actions/resume-session topic-id acp-session-id]]
      [[:acp.actions/new-session topic-id]])))

(defn session-ready
  "Session created/resumed successfully. Save acp-session-id and mark live."
  [state topic-id acp-session-id]
  (js/console.log "[ACP] Session ready:" acp-session-id "for topic:" topic-id)
  [[:loading.actions/set-loading? topic-id false]
   [:effects/save (topic-state/acp-session-id-path topic-id) acp-session-id]
   [:effects/save acp-state/live-topics-path (acp-state/mark-live state topic-id)]])
```
