# Open Any Markdown File — Decouple Storage From Document Folder

**Date:** 2026-05-18
**Branch:** `feat/open-any-markdown-file`
**Status:** Design / Spec (also serves as the brainstorming spec; once approved, copy to `docs/superpowers/specs/2026-05-18-open-any-markdown-file-design.md` and commit before implementation planning.)

---

## Context

Today, a gremllm "workspace" is a folder the user picks. That folder simultaneously contains the artifact (`document.md`, hardcoded name) and gremllm's per-document state (`topics/*.edn`). This conflation works when the user has dedicated a folder to gremllm — but it blocks the goal of letting users open any `.md` file from anywhere on their machine. We can't drop a `topics/` folder next to every markdown file a non-technical business user double-clicks.

This change decouples *the file the user cares about* from *where gremllm keeps its own per-document state*. The user picks any `.md` file from anywhere; gremllm reads/writes its state from `userData`, keyed by the file's absolute path. The user's filesystem is left untouched.

This is an MVP. Rename/move handling, migration of existing workspaces, and a recent-documents UI are explicitly deferred.

The target user is a non-technical business user (PE due-diligence associate as the beachhead). The chosen approach mirrors the model VS Code uses for workspace state (SQLite `workspaceStorage` keyed by path hash) and Bear (central library), which together cover a large, satisfied user base. The sidecar-folder model (Obsidian/Git) was rejected because it pollutes the user's directory tree and surprises users who didn't intentionally create a workspace.

## Storage Layout

```
<userData>/User/                                    # already exists; produced by io/user-dir-path
└── documents/
    └── <sha256(absolute-doc-path)>/
        ├── topics/
        │   └── *.edn        # unchanged shape, just relocated
        └── meta.edn         # { :doc-path "/Users/.../memo.md" }
```

- One directory per document, keyed by the SHA-256 of the document's normalized absolute path. Hashing avoids filename-safety problems for paths with `/`, spaces, or unicode.
- `meta.edn` stores the original document path. Not load-bearing for v1 — included so a future "recent documents" UI is trivial to add.
- The existing topic file format is unchanged; codecs and schema are not touched.

## User Flow Changes

| Today | New |
|---|---|
| Menu: **Open Workspace Folder** → directory picker | Menu: **Open…** → file picker, `.md` / `.markdown` filter |
| Reads `<folder>/document.md` (hardcoded name) | Reads the picked `.md` file at its actual path |
| Reads `<folder>/topics/*.edn` | Reads `<userData>/User/documents/<hash>/topics/*.edn` |
| Workspace folder visible in UI as the project boundary | No workspace concept exposed to the user; "you open a document" is the whole mental model |

The hardcoded `document.md` filename disappears as a side effect — solving the original "part 1" goal.

## IPC Channel Changes

Rename, preserve handler shape:

| Today | New |
|---|---|
| `workspace/pick-folder` | `document/open` |
| `workspace/reload` | `document/reload` (accepts current document path) |
| `workspace:opened` (event) | `document:opened` (event) |

The `document/create` channel is removed — there is no longer an app-mediated "create a document" flow. Users create `.md` files however they normally would (Finder, another editor, etc.) and open them. If we discover this hurts onboarding for the PE user, a "New Document…" save-file dialog can be added later.

**Empty state.** Today, when a workspace has no `document.md`, the document panel shows a "Create Document" button. In the new model, the equivalent state is "no document is open yet" (first launch, or after closing one). The empty-state panel becomes: a muted prompt ("No document open") plus an "Open…" button that triggers the same file picker as the menu item. No "create" affordance in v1.

## Code Changes (Concrete Files)

**Changes:**

- `src/gremllm/main/io.cljs` — remove `document-file-path` (which composed `<workspace>/document.md`). Add `document-storage-dir [user-data-dir doc-path]` → returns `<userData>/User/documents/<hash>/`, and `path->document-hash [doc-path]` (SHA-256 of normalized absolute path, hex-encoded). `topics-dir-path` continues to exist but now takes the document storage dir, not the workspace folder.
- `src/gremllm/main/effects/workspace.cljs` —
  - Replace `pick-folder-dialog` with `pick-document-dialog`: uses `showOpenDialog` with `properties: ["openFile"]` and `filters: [{ name: "Markdown", extensions: ["md", "markdown"] }]`. Dispatches `[:document.actions/open doc-path]` on success.
  - Update `load-and-sync` to take a document path: reads the `.md` file directly from that path, computes the storage dir, loads topics from `<storage-dir>/topics/`, writes `meta.edn` if not present.
  - Remove `create-document`.
- `src/gremllm/main/actions/document.cljs` — delete the file; `create-plan` is the only function and is no longer referenced.
- `src/gremllm/renderer/actions/document.cljs` — remove `create`, `create-success`, `create-error` (no callers after the empty-state UI changes).
- `src/gremllm/main/core.cljs` — rename IPC handlers per the table above. Drop `document/create`.
- `src/gremllm/main/menu.cljs` — menu label change ("Open Workspace Folder" → "Open…"), command name update.
- `src/gremllm/main/state.cljs` — `workspace-directory` now resolves to the document's storage dir (computed from the active document path), not a user-picked folder. Add `active-document-path` lookup.
- `src/gremllm/main/actions/workspace.cljs` — actions renamed/repurposed to operate on a document path rather than a folder path.
- `resources/public/js/preload.js` — rename bridge methods (`openWorkspaceFolder` → `openDocument`, etc.). Drop `createDocument`.
- `src/gremllm/renderer/actions.cljs` and `src/gremllm/renderer/actions/workspace.cljs` — dispatch renamed actions; payload now carries a document path.
- `src/gremllm/renderer/ui/document.cljs` — UI copy update if "workspace" appears in user-facing strings; replace the "Create Document" empty state with an "Open…" empty state that dispatches `[:document.actions/open]`.
- `src/gremllm/main/README.md` — refresh the "Data Storage" diagram and IPC channel list.

**Unchanged:**

- `src/gremllm/schema.cljs`, `src/gremllm/schema/codec.cljs` — topic disk format is identical.
- The entire ACP layer (`src/gremllm/main/effects/acp.cljs`, `src/js/acp/*`, ACP codecs).
- Renderer document/diff rendering (`renderer/ui/document/diffs.cljs`).
- Topic save/delete (`main.actions.topic`, save path inside `workspace.cljs`).

**Naming note (deferred decision):** Per the project's modelarity principle, `workspace` is now a leaky name — it implies a folder boundary that no longer exists in the user model. Internally, `workspace` is kept for this change to minimize churn, but a follow-up rename pass (`workspace` → `document` or `document-context`) should be considered after this lands and the new shape settles.

## Out of Scope (v1)

- **Rename/move handling.** If the user moves `memo.md` in Finder, the chat history is orphaned in `userData`. No detection, no relinking UI. Documented behavior.
- **Migration of existing workspace folders.** Users with an existing `<folder>/topics/` directory will not see those topics in the new model. No automatic import.
- **Recent-documents list / file history UI.** `meta.edn` makes this trivial later.
- **Multi-window / multiple documents open at once.** One document at a time, same as today.
- **Cross-volume / removable-drive considerations.** Path is path; if a path becomes invalid, the document just fails to open.

## Verification

Manual end-to-end (run `npm run dev`):

1. **Open from anywhere:** create a `.md` file at `~/Desktop/test.md` with some content. App → **File → Open…** → pick it. Document panel renders the content. ✅
2. **Chat persists per document:** start a topic, send a message, close the app. Reopen the same `.md`. Topic and message history are restored. ✅
3. **Different documents have independent state:** create `~/Desktop/a.md` and `~/Desktop/b.md`. Open `a.md`, start a topic. Open `b.md` — no topics. Open `a.md` again — topic is back. ✅
4. **User folder stays clean:** after step 1–3, `~/Desktop/` contains only the two `.md` files. No `topics/`, no `.gremllm/`. ✅
5. **Storage location verified:** inspect `<userData>/User/documents/` (on macOS: `~/Library/Application Support/gremllm/User/documents/`). One subdir per document opened, each containing `topics/` (if topics created) and `meta.edn` with the original path. ✅
6. **Orphan behavior is acceptable (out-of-scope confirmation):** rename `~/Desktop/a.md` to `~/Desktop/a-renamed.md` in Finder. Open `a-renamed.md` in app. No topics appear (expected — this is the deferred case). The old hash directory is still on disk. ✅

Automated:

- Unit tests for `path->document-hash` (stable, hex, fixed-length, different paths → different hashes).
- Unit tests for `document-storage-dir` composition.
- Existing `enumerate` / `load-topics` tests work unchanged (they take a topics dir; what changes is who composes that path).
- `npm run test:all` should pass with no regressions in topic save/load, ACP, or codec layers.
